## はじめに

IPv6のアドレス体系、ちゃんと理解できていますか？

AWS VPCのデュアルスタック対応、CloudflareのIPv6設定、Linuxサーバーのネットワーク設定など、インフラエンジニアがIPv6に触れる場面は増えています。しかしIPv4と比べてアドレスが長く、種類も多いため「なんとなく分かるけど正確には説明できない」という方も多いのではないでしょうか。

この記事では、IPv6アドレスの種類をIPv4との対応関係を軸に整理します。IPv4の知識がある方なら、対応づけて覚えることでスムーズに理解できるはずです。

## IPv6アドレスの基本

### アドレスの長さと表記

IPv4は32ビット（例：`192.168.1.1`）ですが、IPv6は128ビットで4倍の長さがあります。
IPv6は16進数で表記し、4桁ずつコロンで区切ります。

```
2001:0db8:0000:0000:0000:0000:0000:0001
```

### 省略ルール

IPv6にはアドレスを短く書くための省略ルールが2つあります。

**ルール1：各グループの先頭の0は省略できる**

```
2001:0db8:0000:0000:0000:0000:0000:0001
↓
2001:db8:0:0:0:0:0:1
```

**ルール2：連続する0のグループは `::` で省略できる（1箇所のみ）**

```
2001:db8:0:0:0:0:0:1
↓
2001:db8::1
```

`::` を2回以上使うとどこが省略されたか分からなくなるため、使えるのは1箇所だけです。この制約は設定ミスの原因になりやすいので、覚えておきましょう。

## プレフィックス長（/の後ろの数字）

IPv4のサブネットマスクと同じ概念です。「先頭から何ビットがネットワーク部か」を示します。

- IPv4の `/24` → 先頭24ビットがネットワーク部、残り8ビットがホスト部
- IPv6の `/64` → 先頭64ビットがネットワーク部、残り64ビットがホスト部

AWSのVPC設計でCIDRを指定するときも、このプレフィックス長の考え方はIPv4・IPv6どちらでも同じです。

## IPv6アドレスの種類とIPv4との対応関係

IPv4とIPv6の間に「変換計算式」があるわけではありません。「IPv4でこの役割だったものが、IPv6ではこのアドレス範囲に割り当てられた」という対応関係で覚えます。

| 役割 | IPv6アドレス範囲 | IPv4での相当アドレス | 用途 |
|------|-----------------|-------------------|------|
| ループバック | `::1/128` | `127.0.0.1` | 自分自身への通信テスト |
| リンクローカル | `fe80::/10` | `169.254.0.0/16`（APIPA） | 同一セグメント内のみの通信 |
| ユニークローカル | `fc00::/7`（実際は`fd00::/8`〜） | `10.0.0.0/8`, `192.168.0.0/16` 等 | 組織内部用のプライベートアドレス |
| グローバルユニキャスト | `2000::/3` | パブリックIP | インターネット上の通信 |
| マルチキャスト | `ff00::/8` | `224.0.0.0/4` | 一対多の通信 |
| サイトローカル | `fec0::/10` | プライベートアドレス | **廃止済み**（ユニークローカルに置き換え） |

### 先頭の文字で判別する

IPv6はアドレスの先頭を見るだけで種類を判別できます。これは設定ファイルの確認やトラブルシューティングの際に役立ちます。

- **`::1`** → ループバック
- **`fe80`** で始まる → リンクローカル
- **`fc` または `fd`** で始まる → ユニークローカル（プライベート）
- **`2000`** 台 → グローバルユニキャスト
- **`ff`** で始まる → マルチキャスト

## リンクローカルアドレスの注意点

リンクローカルアドレスには、インフラエンジニアが知っておくべき重要なポイントがあります。

**`fe80::/10`** は仕様上の定義です。先頭10ビットが `1111111010` であるアドレス全てがリンクローカルと定義されています。

**`fe80::/64`** は実際にインターフェースに設定されるときのプレフィックス長です。現実のネットワークでは先頭64ビットをネットワーク部として使うため、リンクローカルアドレスは `fe80::` の後にインターフェースID（通常はMACアドレスから生成）が入ります。

Linuxサーバーで `ip addr` を実行すると、ネットワークインターフェースに `fe80::` で始まるアドレスが自動で割り振られているのを確認できます。これがリンクローカルアドレスです。

```bash
$ ip addr show eth0
    inet6 fe80::xxxx:xxxx:xxxx:xxxx/64 scope link
```

## 実務での活用場面

### AWS

AWSではVPCにIPv6 CIDRブロックを割り当てることができます。IPv6を有効にしたVPCではデュアルスタック（IPv4とIPv6の両方）で運用され、EC2インスタンスにはグローバルユニキャストアドレス（`2000::/3`の範囲）が割り当てられます。

セキュリティグループやネットワークACLでIPv6のルールを設定する際にも、アドレスの種類を理解しておく必要があります。

### Cloudflare

CloudflareはデフォルトでIPv6に対応しており、オリジンサーバーへの接続もIPv6で行われることがあります。DNSレコードにAAAAレコード（IPv6用のAレコード）を設定する場面でも、IPv6アドレスの表記を正しく理解していることが前提になります。

### Linuxサーバー

Linuxではデフォルトでインターフェースにリンクローカルアドレスが自動設定されます。IPv6が不要な環境で無効化する場合や、逆にIPv6でサービスをリッスンさせる設定を行う場合にも、アドレス体系の知識が必要です。

## まとめ

IPv6のアドレス体系は、IPv4との対応関係をセットで覚えるのが効果的です。

- ループバック：`::1/128`（= `127.0.0.1`）
- リンクローカル：`fe80::/10`（= `169.254.0.0/16`）
- ユニークローカル：`fc00::/7`（= プライベートアドレス）
- グローバル：`2000::/3`（= パブリックIP）
- マルチキャスト：`ff00::/8`（= `224.0.0.0/4`）

先頭の文字で種類を判別できるようにしておけば、設定ファイルの確認やトラブルシューティングの場面で素早く対応できます。

IPv6はこれからますます避けて通れない技術になっていきます。基礎をしっかり押さえて、IPv4の知識をIPv6にも広げていきましょう。
